# 一、什么是高阶函数（Higher-Order Functions）

在函数式编程中：

> **高阶函数 = 接收函数作为参数 或 返回函数的函数**

Swift 是一门 **多范式语言**，其标准库大量采用函数式设计，因此高阶函数几乎贯穿整个 Swift API。

典型特征：

- 函数作为参数
- 函数作为返回值
- 闭包（Closure）驱动
- 不可变数据 + 转换思想


# 二、为什么 Swift 极度依赖高阶函数

Swift 标准库核心协议：

```swift
Sequence
Collection
BidirectionalCollection
RandomAccessCollection
```

几乎所有集合操作都建立在：

```
(sequence) -> transform -> new sequence
```

这种 数据流（Data Pipeline）模型 上。

## Imperative vs Functional

传统写法：

```
var result: [Int] = []
for i in numbers {
    if i % 2 == 0 {
        result.append(i * 2)
    }
}
```

函数式：

```
let result = numbers
    .filter { $0 % 2 == 0 }
    .map { $0 * 2 }
```

优势：

- 声明式
- 可组合
- 可测试
- 可并行优化（未来潜力）

# 四、map —— 数据转换的核心

## 定义

```
func map<T>(_ transform: (Element) -> T) -> [T]
```

## 本质

```
Element -> Transform -> NewElement
```

## 示例

```
let nums = [1,2,3]
let squares = nums.map { $0 * $0 }
```

## 高级用法：KeyPath Map

```
struct User {
    let name: String
}

let users = [User(name: "name1"),User(name: "name2")]
let names = users.map(\.name)
print(names)
```

打印结果：

```
["name1", "name2"]
```

✅ 编译期优化

✅ 更少闭包开销

# 五、compactMap —— Optional 消除神器

## 设计目标

**解决：**

```
[T?] -> [T]
```

## 示例

### 数组
```
let strings = ["1", "2", "a"]

let numbers = strings.compactMap(Int.init)

//>也可以使用闭包，在这里闭包里处理更加灵活
let numbers1 = strings.compactMap { value in
    Int(value)
}


print(numbers)
```

打印结果：

```
[1, 2]
```

### 字典

```
let dict = [
    "a": "1",
    "b": "2",
    "c": "x"
]
let dicNum = dict.compactMap { (key, value) in
    Int(value)
}

print(dicNum)
```

打印结果：

```
[2, 1]
```

**如果需要返回字典——>使用compactMapValues**


```
let dict = [
    "a": "1",
    "b": "2",
    "c": "x"
]
let dicNum1 = dict.compactMapValues(Int.init)

print(dicNum1)
```

打印结果：

```
["b": 2, "a": 1]
```




## 内部逻辑：

```
if let value = transform(element) {
    append(value)
}
```

## 应用场景
- JSON 解析
- URL 构造
- 数据清洗


# 六、flatMap —— 扁平化的真正含义

**很多开发者误解 flatMap。**

作用：

```
[[T]] -> [T]
```

示例：

```
let array = [[1,2],[3,4]]
let result = array.flatMap { $0 }
```

结果：

[1,2,3,4]

### Optional FlatMap（极其重要）

定义（简化）

```
func flatMap<U>(_ transform: (Wrapped) -> U?) -> U?
```

注意区别：

```
map     : T -> U

flatMap : T -> U?
```

### Optional使用map vs flatMap 的本质区别

- 🔥使用map时->出现 Optional 嵌套
- ✅ 使用flatMap时->自动“拍平”(flatten)

示例：

```
let str: String? = "123"
let result1 = str.map { Int($0) }
let result2 = str.flatMap { Int($0) }
print(result1)
print(result2)
```

结果：

```
Optional(Optional(123))
Optional(123)
```

### 应用场景：JSON 安全解析

**传统写法：**

```
let json: [String: Any] = ["age": "18"]

if let strAge = json["age"] as? String {
    if let intAge = Int(strAge){
        ///> 处理值
        print(intAge)
    }
}
```


**flatMap 写法：**

```
let json: [String: Any] = ["age": "18"]

json["age"]
    .flatMap { $0 as? String }
    .flatMap(Int.init)
    .map { age in
        ///> 处理值
        print(age)
    }
```

### Optional.flatMap vs Sequence.flatMap（重点）


**Optional.flatMap**

- 拍平 Optional
- T?? → T?

**Array.flatMap**
- [[T]] → [T]

# 八、reduce —— 最强但最容易滥用

## 定义

```
func reduce<Result>(
    _ initialResult: Result,
    _ next: (Result, Element) -> Result

```

- initialResult: 初始值
- next：遍历执行某种操作


## 求和

```
let numbers = [1,2,3,4,5]
let sum = numbers.reduce(0) { result, num in
    result + num
}

```

```
let sum = numbers.reduce(0, +)
```


## reduce 的真正能力

它可以构造 任何结构：

```
let dict = users.reduce(into: [Int:User]()) {
    $0[$1.id] = $1
}
```


# 九、joined —— 序列扁平化与字符串拼接的高级用法

`joined` 是 Swift 高阶函数中**非常容易被低估**的一个，它本质上属于：

> **Sequence Flattening（序列展开）**

很多开发者只把它当字符串拼接工具，但实际上它是一个 **惰性扁平化操作**。

---

## joined 的两种核心形态

###  1.扁平化序列（Flatten Sequence）

#### 定义：

```swift
func joined() -> FlattenSequence<Self>
where Element : Sequence
```

#### 含义：

```
Sequence<Sequence<T>> → Sequence<T>
```

#### 示例:

```
let array = [[1,2],[3,4]]

let result = array.joined()
```

#### 类型：

```
FlattenSequence<[[Int]]>
```

#### 注意：

⚠️ 结果不是 [Int]

而是一个惰性序列。

如果需要数组：

```
let result = Array(array.joined())
```

**关于惰性序列，后面lazy会详细介绍**



###  2.字符串专用 joined(separator:)

这是大家最熟悉的版本。

#### 定义：

```
func joined(separator: String) -> String
where Element == String
```

#### 示例：

```
let words = ["Swift", "is", "awesome"]

let sentence = words.joined(separator: "-")
```

#### 结果：

```
Swift-is-awesome
```

# 十、contains

`contains` 是 `Swift Sequence` 中 非常核心的查找 API，但它其实有 3 种完全不同的用法，很多人只用过第一种。

## 1.最基础：判断是否包含元素

```
let arr = [1,2,3,4]
arr.contains(3) // true
```

定义：

```
func contains(_ element: Element) -> Bool
where Element: Equatable
```

### 要求

元素必须实现 `Equatable`。

### 自定义类型示例

```
struct User: Equatable {
    let id: Int
}

let users = [User(id: 1), User(id: 2)]

users.contains(User(id: 1)) // true
```

对于继承自NSObject的类，由于`NSObject`已经遵循了`Equatable`协议，因此重写`isEqual`就可以实现`contains`

```
class User:NSObject{
    let id: Int
    init(id: Int) {
        self.id = id
    }
    
    override func isEqual(_ object: Any?) -> Bool {
        guard let object = object as? User else{
            return false
        }
        return id == object.id
    }
}

let users = [User(id: 1), User(id: 2)]

users.contains(User(id: 1)) // true
```

## 2.结合闭包使用

这是实际开发中更常用的：

```
contains(where:)
```

### 示例

```
let arr = [1,2,3,4]

let result = arr.contains { $0 > 3 }
print(result) // true
```

定义：

```
func contains(
    where predicate: (Element) throws -> Bool
) rethrows -> Bool
```

### 等价逻辑（理解本质）

```
for e in arr {
    if predicate(e) {
        return true
    }
}
return false
```

👉 找到第一个就停止（短路）

## 3.性能关键点

### `contains` 是：

✅ O(n)

✅ 短路执行

✅ 不会遍历全部

----

### `contains` 对比` filter`

示例： 判断数组中有没有大于3的元素

#### ` filter`版本：

```
arr.filter { $0 > 3 }.isEmpty == false
```

执行：

```
遍历全部元素
创建新数组
```

#### ` contains `版本：

```
arr.contains { $0 > 3 }
```

执行：

```
找到即停止
无新数组
```

这种场景推荐使用` contains `

# 十一、allSatisfy

allSatisfy 是 Swift Sequence 中一个非常优雅的 API，用来判断：

```
序列中的所有元素是否都满足某个条件
```

很多人用 filter 或 contains 实现，其实都不如 allSatisfy 直接。

## 1.基本用法

```
let arr = [2,4,6,8]

let result = arr.allSatisfy { $0 % 2 == 0 }

print(result) // true
```

含义:

```
所有元素都是偶数吗？
```

## 2. 底层逻辑（理解本质）

`allSatisfy` 的实现非常简单：

```
for element in sequence {
    if !predicate(element) {
        return false
    }
}
return true
```

特点：

- 短路执行

- 只要有一个不满足就停止

## 3. 和 contains 的关系

这两个 API 实际是 **对偶关系**：

###contains

存在一个满足条件

```
arr.contains { $0 > 5 }
```

###allSatisfy

全部满足条件

```
arr.allSatisfy { $0 > 0 }
```

等价转换

```
arr.allSatisfy { condition }
```

等价于：

```
!arr.contains { !condition }
```

例子：

```
arr.allSatisfy { $0 > 0 }
```

等价：

```
!arr.contains { $0 <= 0 }
```

## 4. 特殊注意：空数组的特殊情况

### 示例：

```
[].allSatisfy { $0 > 0 }
```

### 结果：

``
true
``

### 原因：

数学逻辑叫：`Vacuous Truth（空真）`

意思：没有元素违反条件,所以为 true。


# 十二、sorted —— 排序的函数式表达

`sorted` 是 Swift 高阶函数体系中 **排序（Ordering Transformation）** 的核心 API。

很多开发者只把它当排序工具，但从函数式角度看：

> `sorted` = **基于规则的序列重排（Reordering Transformation）**

它不会修改原集合，而是返回一个新的序列结果。

---

## 1.sorted 的定义

```swift
func sorted(
    by areInIncreasingOrder: (Element, Element) -> Bool
) -> [Element]
```

关键点：

- 接收一个函数（闭包）

- 返回新数组

- 原数据保持不变（值语义）

### 基础示例

```
let numbers = [3,1,4,2]

let result = numbers.sorted { $0 < $1 }
```

结果：

```
[1,2,3,4]
```

## 2.sorted vs sort

|API|是否修改原数组|返回|
|---|---|---|
|sorted	|❌| 新数组|
| sort	| ✅ |Void|

### sorted（函数式）

```
let newArray = array.sorted()
```

特点：

- 不可变风格

- 线程安全

- 链式调用友好

### sort（命令式）

```
array.sort()
```

特点：

- 原地修改

- 无额外分配

- 性能略优

### ✅ 经验法则

```
Pipeline → sorted
性能关键路径 → sort
```

## 3.排序闭包的真正含义

闭包：

```
{ a, b in a < b }
```

不是：

```
a 是否排在 b 前面？
```

而是：

```
a 应该在 b 之前吗？
```

这是一个 **Strict Weak Ordering** 判断函数。

### 降序

```
numbers.sorted { $0 > $1 }
```

### 使用运算符函数（推荐）

```
numbers.sorted(by: <)
numbers.sorted(by: >)
```

## 4. KeyPath 排序

```
struct User {
    let name: String
    let age: Int
}
```

传统：

```
users.sorted { $0.age < $1.age }
```

### ✅ 更现代写法（Swift KeyPath）

可以扩展Sequence
```
extension Sequence {

    func sorted<T: Comparable>(
        by keyPath: KeyPath<Element, T>
    ) -> [Element] {
        sorted { $0[keyPath: keyPath] < $1[keyPath: keyPath] }
    }
}
```

使用：

```
users.sorted(by: \.age)
```

优势：

- 可读性高

- 可复用

- 减少闭包噪音

# 十四、lazy 惰性序列的设计

## 1.什么是 lazy？

在 Swift 中：

```swift
sequence.lazy
```

并不是执行某个操作，而是：

> **将 Sequence 转换为 LazySequence，使后续高阶函数延迟执行（Lazy Evaluation）。**

## 2.对比：Eager vs Lazy

### 普通（Eager）

```
let result = array
    .map { f($0) }
    .filter { g($0) }
```

执行流程：

```
map 遍历全部 → 生成新数组
filter 再遍历 → 再生成新数组
```

时间复杂度遍历次数：

```
2 × N
```

中间数组：✅ 产生

### Lazy

```
let result = array
    .lazy
    .map { f($0) }
    .filter { g($0) }
```

执行方式：

```
元素1 → map → filter → 输出
元素2 → map → filter → 输出
```

遍历次数：

```
1 × N
```

中间数组：❌ 不产生

## 3.lazy 本质不是函数

### lazy 改变的是 Sequence 类型

**类型变化**

```
Array<Int>
    ↓ lazy
LazySequence<Array<Int>>
```

从这里开始：

后续所有支持 lazy 的操作都会返回 LazyXXXSequence。

## 4. Lazy Pipeline 的类型演化

```
let result =
    array.lazy
        .map { $0 * 2 }
        .filter { $0 > 10 }
        .map { "\($0)" }
```

真实类型（简化）：

```
LazyMapSequence<
    LazyFilterSequence<
        LazyMapSequence<
            LazySequence<Array<Int>>
        >
    >
>
```

Swift 构建的是：

>计算管道（Computation Pipeline）

而不是数据。

## 5. 自动转换

```
var words = ["Swift", "is", "awesome"]

words = words.lazy.map { $0 + $0 }
```

`words` 类型：

```
[String]
```

右侧是` LazySequence`。

Swift 自动执行：

```
words = Array(lazySequence)
```
